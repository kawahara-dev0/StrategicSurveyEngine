# Strategic Survey Engine

Survey feedback platform with schema-per-survey isolation (backend + frontend in one repo).

## Design & specs

- **`.kiro/base`** – Design documents for this project. Vibecoding is performed based on these specs.
- **`.kiro/specs/strategic-survey-en`** – Design documents generated by an AI agent (Cursor) to guide implementation.

## Project layout

```
StrategicSurvey/
├── backend/       # FastAPI + SQLAlchemy 2.0 (Admin API, schema switching)
├── frontend/      # React 18 (Vite) + TypeScript + Tailwind
└── docker-compose.yml   # DB + Backend + Frontend
```

---

## Quick start with Docker Compose

Runs PostgreSQL, backend (FastAPI), and frontend (Vite dev server). Migrations run on backend startup.

```bash
docker compose up -d
```

- **Frontend**: http://localhost:5173
- **API docs**: http://localhost:8000/docs
- **Health**: http://localhost:8000/health

Copy `.env.example` to `.env` at project root and set `ADMIN_API_KEY`. Docker Compose reads the root `.env`.

```bash
docker compose down   # stop all
```

## Environment variables

Use the **project root `.env`** for Docker Compose and local dev. Copy `.env.example` to `.env` and set values as needed.

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | PostgreSQL URL (`postgresql+asyncpg://...` for backend). |
| `ADMIN_API_KEY` | Optional; when set, required as `X-Admin-API-Key` for Admin API. |
| `JWT_SECRET_KEY` | Manager auth JWT signing key. Default used if unset. Production: use 32+ byte random value. |
| `VITE_API_URL` | Frontend API base. Default `/api` (Vite proxy). With Docker: `http://localhost:8000`. |

## Backend

- **Stack**: FastAPI, SQLAlchemy 2.0 (async), PostgreSQL 15+, Alembic.
- Schema-per-survey; Admin API, Contributor Submission API, Moderation API, Manager API (JWT).

### Local setup (without Docker)

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate   # Windows
# source .venv/bin/activate   # Linux/macOS
pip install -r requirements.txt
```

Ensure project root `.env` exists with `DATABASE_URL`. Then:

```bash
python -m alembic upgrade head
uvicorn app.main:app --reload
```

### Test (pytest)

Requires PostgreSQL and migrations. Run inside Docker (recommended if no local Python):

```bash
docker compose exec backend pytest tests/ -v
```

Or locally with Python 3.11+:

```bash
cd backend
pip install -r requirements-dev.txt
alembic upgrade head
pytest tests/ -v
```

### Lint & type check

```bash
pip install -r requirements-dev.txt
ruff check . && ruff format --check .
mypy app
```

### API overview

| API | Auth | Examples |
|-----|------|----------|
| Admin | `X-Admin-API-Key` | POST /admin/surveys, POST /admin/surveys/{id}/questions |
| Contributor | none | GET /survey/{id}/questions, POST /survey/{id}/submit |
| Moderation | Admin | GET/POST /admin/surveys/{id}/responses, /opinions |
| Manager | JWT (Bearer) | POST /manager/auth → JWT, then /manager/{id}/opinions, export |

## Frontend

- **Stack**: React 18, Vite, TypeScript, Tailwind, TanStack Query, React Router.
- Survey list, create survey (with one-time Access Code), survey detail, contributor submission form (`/survey/:id/post`), Manager dashboard.

### Local setup

1. Start the backend.
2. From project root:

```bash
cd frontend
npm install
npm run dev
```

Vite reads `.env` from project root. For local dev, default `VITE_API_URL=/api` uses the Vite proxy to the backend.

### Build

```bash
npm run build
npm run preview
```

### Lint & test

```bash
npm run lint
npm run format:check
npm test   # Jest, mocks API (backend not required)
```
